# Tables

_Last updated: 2026-03-08_

## Purpose

The Tables page is for fast inspection of table data, profiling, paging, and quick data-oriented checks.

## What You Can Do Here

- Browse table/view data
- Search and filter visible records
- Switch between data and profiling perspectives
- Inspect schema-related context while exploring records

## How to Use It

1. Open `Tables`.
2. Select a table or view.
3. Use search/filter controls to narrow results.
4. Use paging controls for large datasets.
5. Switch to profiling when you need quality insights.

## Tips

- Start broad, then narrow filters to avoid missing context.
- Use profiling to detect null-heavy or inconsistent columns early.
- Move to SQL Workspace when you need custom joins or transformations.
